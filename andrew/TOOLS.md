## Project Onboarding Protocol

Before starting any new project with Andrew, always offer these two options:

1. **Git Clone** — Clone the repo to `projects/<name>` in workspace (preferred for version-controlled projects)
2. **Copy/Mirror** — Copy the project folder to `projects/<name>` in workspace (for non-git projects or when a sandbox is needed)

Either way, the work happens IN the workspace so I can:
- Track changes and context
- Maintain continuity across sessions
- Log progress to memory files

Never work directly in external directories for substantial projects.
